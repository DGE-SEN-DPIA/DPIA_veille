## ⚡ À retenir (importance 3)

### DeepSeek V4 — GA officielle ce jour (15/07) [SUITE, fait nouveau : date GA confirmée]
- **Fait :** DeepSeek V4 Pro et V4 Flash passent de preview (depuis 24/04/2026) à production officielle **ce 15 juillet 2026**. Nouveautés par rapport à la preview : contexte standard **1 million de tokens** sur toute la gamme ; performances améliorées en agentique, maths et génération de code. Changement structurel le plus impactant : **instauration du peak-hour pricing**, première dans l'industrie LLM — ×2 aux heures de pointe Beijing (9h-12h et 14h-18h, heure locale). Tarifs off-peak : V4-Pro $0,435/$0,87 par M tokens (in/out), V4-Flash $0,14/$0,28. **Deadline migration API : 24 juillet 2026, 15h59 UTC** — les noms d'API `deepseek-chat` et `deepseek-reasoner` seront désactivés sans période de grâce (migration obligatoire en 9 jours).
- **Impact opérationnel :** Le pricing peak/off-peak est inédit pour un fournisseur LLM frontière et complexifie les architectures serverless à trafic variable (coût facture inconnu à l'avance). Pour toute organisation française ayant intégré DeepSeek en API — via Scaleway, OVH ou accès direct — : vérifier d'urgence les noms de modèles appelés, calculer l'impact tarifaire peak/off-peak et migrer avant le 24 juillet.
- **Sources :** DeepSeek API Docs, api-docs.deepseek.com/news/news260424/ [source primaire] ; TechNode, technode.com, 30/06/2026 ; ExplainX, explainx.ai/blog/deepseek-v4-official-release-peak-pricing-mid-july-2026 [2 sources concordantes].

### Grok Build CLI — fuite de données dépôts Git (14/07) [NOUVEAU]
- **Fait :** Une analyse wire-level publiée le 12 juillet a prouvé que le CLI de **Grok Build** (xAI/SpaceX) packageait l'intégralité des dépôts Git suivis — historique complet, secrets effacés inclus — et les transmettait à un bucket **Google Cloud Storage**, sans consentement explicite de l'utilisateur. L'analyse a atteint la une de Hacker News le **14 juillet**. Réaction d'Elon Musk (14/07, post X) : promesse de « suppression complète et totale » de toutes les données uploadées avant le 13 juillet. À ce jour : aucun calendrier publié, aucun mécanisme permettant à un utilisateur de vérifier que ses données ont été supprimées, aucun bilan du nombre d'organisations affectées, aucune notification individuelle annoncée.
- **Impact sécurité DSI :** Tout dépôt utilisé avec Grok Build CLI avant le 13 juillet doit être traité comme potentiellement compromis — en particulier si l'historique Git contenait des clés API, tokens cloud, mots de passe base de données, clés SSH ou webhooks secrets ayant été commités puis supprimés. Action immédiate : rotation de tous les secrets Git des dépôts concernés, audit de l'historique, revue des politiques d'autorisation des outils de développement IA.
- **Sources :** TechTimes, « Grok Build Shipped Entire Codebases to xAI Cloud; Privacy Toggle Did Nothing », techtimes.com, 14/07/2026 ; Hacker News, 14/07/2026 [front page, 2 sources concordantes].
